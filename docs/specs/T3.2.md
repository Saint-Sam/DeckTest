# T3.2 Legacy Script Parser Specification

Status: implementation verification

## Scope

`forge-porttools::legacy` parses every vendored legacy Forge card script into a
positioned, source-ordered legacy AST. Parsing is structural and deliberately
does not claim that an ability can already lower into the Forge 2.0 typed IR;
that mapping belongs to T3.3.

The parser recognizes:

- `A:` activated abilities;
- `T:` triggered abilities;
- `R:` replacement effects;
- `S:` static abilities;
- `K:` keywords with colon-separated arguments;
- `SVar:<name>:<value>` declarations;
- `ALTERNATE` multi-face boundaries; and
- all other key/value metadata as preserved properties.

Ability and SVar expressions retain the original trimmed text and split every
pipe field at its first `$` separator. Unknown selectors, API names, parameter
keys, and metadata remain data in the legacy AST so T3.3 can classify or
quarantine them without losing source information.

Blank lines and full-line `#` comments are ignored. Every other line must be a
recognized marker or contain a nonempty key followed by `:`. Empty ability and
keyword payloads, malformed SVar declarations, and unstructured lines fail
with stable reason codes and one-based line/column diagnostics.

## Corpus Gate

The local audit recursively sorts and parses all `.txt` files under the pinned
vendored `cardsfolder`. It binds metrics to the exact local submodule revision,
emits deterministic metrics and a reason-coded
failure list. The gate requires at least 99.5% file-level parse coverage, runs
without network access, and fails if the committed metrics become stale.

Current pinned-corpus result:

- 33,290 scripts discovered;
- 33,290 scripts parsed;
- 0 failed scripts;
- 100.0000% parse-only coverage;
- 296,879 positioned non-comment AST lines; and
- no lossy UTF-8 decoding.

This result proves syntax coverage only. API lowering, smoke execution, and
card-specific behavior are later T3.3-T3.6 gates.

The recurring T3 integration lane recompiles deterministic card databases,
validates the catalog, runs compiled nightmares and all semantic Oracles, and
regenerates workspace coverage. The signed CP-DSL mutation, sanitizer, and
cross-platform packet remains bound to its exact checkpoint commit and is not
recreated for every parser or mapper integration.
