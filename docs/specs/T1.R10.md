# T1.R10 Prompted Human Play

## Purpose

T1.R10 exposes the production four-player Commander path as a real terminal
game. One Owner-controlled seat chooses numbered legal actions while three
deterministic baseline seats use the controller proven by
CP-FOUR-PLAYER-POD.

This task replaces no kernel or card-runtime behavior. It is a control surface
over the same deck loader, card programs, casting, priority, triggers, combat,
commander rules, elimination, invariants, and replay path used by T3.9.

## Decision Boundary

- A decision source receives the acting seat's redacted `PlayerView`, the
  canonical typed `DecisionContext`, and presentation labels derived in the
  exact sorted option order.
- Opponent hand identities and every library identity remain hidden.
- Every listed engine action is checked against a cloned state before display.
- Game mutation continues to cross `forge_core::apply`; no post-start direct
  state mutation is permitted.
- Human and AI main-phase, attacker, blocker, and commander-zone paths consume
  the same canonical option sets; selecting an ID outside the supplied context
  fails closed.

## Replay Contract

`forge-human-play-replay-v1` records:

- manifest, seed, turn limit, and human seat;
- every canonical context ID, `DecisionStateKey`, `PlayerViewHash`, complete
  typed legal-action descriptors, and selected action ID;
- the exact display option list and selected index for UI replay, plus the
  legacy redacted-view fingerprint for backward compatibility;
- every human and baseline typed action with before/outcome/after hashes; and
- the expected winner, turns, final life totals, metrics, and final hash.

Replay regenerates each prompt from the redacted view, verifies the canonical
context and selected action membership, reruns the controller, and directly
reapplies the full typed action stream. Legacy v1 records without the additive
canonical fields retain their original exact-label check. Any mismatch fails
closed.

## Exit And Next Work

Automated scripted-input tests prove readiness but do not satisfy the human
checkpoint. T1.R10 exits only after the Owner completes a real game and replays
it at CP-HUMAN-PLAY-CLI.

After that checkpoint, work proceeds through T4.1-T4.3 and then T4.4+. T4 may
reopen T3 only when it identifies a concrete shared runtime blocker. Such a
blocker receives a small targeted T3 fix and semantic regression test before T4
resumes; broad mapper expansion remains closed.
