# T3.6-A Commander semantic-candidate freeze

## Scope

T3.6-A freezes the first 100 Commander-priority identities that are ready to
enter card-specific semantic test authoring. The public artifact is
`assets/t3_6_commander_semantic_candidates.json`.

**Selection is not semantic verification.** Every selected record remains
`candidate_only`. This freeze does not set `semantic_verified`, prove card
fidelity, or satisfy `CP-CARD-SEMANTICS-100`.

## Inputs and eligibility

`tools/freeze_t3_6_commander_semantics.py` consumes only these local repository
sources:

- `assets/coverage_priority.txt`: authoritative tier and rank order;
- `assets/card_catalog.json`: Oracle identity and scope classification;
- `metrics/priority_coverage.json`: per-priority translation outcome and path;
- `metrics/translation.json`: translation source revision, fingerprint, and
  aggregate counts; and
- emitted legacy card scripts under the translation report's `source_root`.

An entry is eligible only when it resolves to exactly one in-scope catalog
identity, is `emitted` in the priority translation report, has a matching local
legacy source, and matches an explicit stratum predicate. Input schemas,
cross-counts, tier order, source paths, script names, and quotas fail closed.

## Frozen strata

| Allocation | Stratum | Quota |
| ---: | --- | ---: |
| 1 | Stack interaction and protection | 8 |
| 2 | Targeted permanent interaction | 14 |
| 3 | Land ramp and search | 10 |
| 4 | Card flow | 14 |
| 5 | Library and graveyard access | 10 |
| 6 | Token and sacrifice | 10 |
| 7 | Triggered synergies | 10 |
| 8 | Continuous, combat, and costs | 10 |
| 9 | Mana sources | 14 |

The tool records the exact machine predicate for each stratum in the manifest.
Allocation is narrow-to-broad in the table order. Within a stratum, the lowest
repository priority rank wins. Final freeze ranks return to global repository
priority order. Quotas must total and select exactly 100 unique Oracle
identities.

## Exclusions and integrity

Every non-selected priority entry is retained with one stable reason code:
catalog identity missing/out of scope/ambiguous, translation not emitted,
duplicate Oracle identity, no stratum match, or higher-priority cards filled
the matching quota. Translation exclusions retain the upstream translation
reason code without copying diagnostic prose.

The manifest binds the generator, priority list, catalog, priority translation
results, translation summary, and the aggregate eligible-script set with
SHA-256. Selected cards also bind their individual legacy script hash. The
manifest's `payload_sha256` covers every field other than itself.

## Local verification

```sh
python3 tools/freeze_t3_6_commander_semantics.py --self-test
python3 tests/t3_6/test_freeze_commander_semantics.py
python3 tools/freeze_t3_6_commander_semantics.py --check
```

The next stage must author and execute card-specific expected behavior through
production actions with deterministic replay and reason-coded failures. Only a
passing `CP-CARD-SEMANTICS-100` may promote these identities semantically.
