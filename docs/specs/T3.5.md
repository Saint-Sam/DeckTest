# T3.5 Capability-Specific Runtime Smoke

Status: data-driven interpreter slice implemented; T3.5 remains in progress

## Contract

`forge-testkit runtime-smoke <FILE-OR-DIRECTORY> [--report PATH] [--quiet]`
parses compiler-valid `.frs` definitions and emits one of three typed results:
`passed`, `unsupported_setup`, or `failed`. Unsupported setup is excluded from
the pass count and makes the command exit nonzero. Compiler, file, production
action, postcondition, and invariant errors are failures rather than
unsupported results.

Card behavior is compiled by `forge-cards::runtime` into an immutable typed
program. The testkit builds a legal deterministic game state, supplies explicit
targets and choices, and executes the program entirely through production
`forge-core` actions. All bindings and dynamic values are resolved and
validated before the first state mutation. Unsupported IR fails closed with a
stable path-qualified diagnostic; no unsupported operation becomes a no-op or
pass.

The current interpreter surface covers:

- normal instant, sorcery, land, and permanent programs with ordinary printed
  mana costs;
- permanent-spell resolution and base object characteristics;
- life gain/loss, card draw, scry, library shuffle, permanent destruction,
  object exile, stack-entry counters, and typed zone movement;
- player, opponent, permanent, owned-zone object, and stack-entry targets;
- target predicates for controller, owner, required/forbidden types, and
  artifact-or-enchantment unions;
- literal amounts and target-power-derived amounts.

The synthesizer creates players and opening hands, reaches legal cast windows,
pays exact costs, casts through `Action::CastSpell`, resolves production
priority, and checks the expected destination. It supplies deterministic legal
objects and stack entries for target requirements. After every successful
action it checks zone conservation, allocated versus streaming hashes, player
views, life/poison sanity, and relevant capability postconditions.

## Current Local Evidence

All commands ran locally with Cargo `--offline`; no network, install, GitHub
Actions, push, or PR was used.

- Focused tests: `forge-core` 96 passed, `forge-cards` 10 passed, and
  `forge-testkit` 28 passed across library and binary targets; doc tests passed.
- Corpus audit: 20,082 translated definitions classified, 515 passed runtime
  smoke, 19,567 were reason-coded unsupported, and 0 failed. The previous
  bounded baseline was 379 passes, so this slice adds 136 real executions.
- Passing capability counts are 342 permanent spells, 72 destruction, 51 draw,
  39 life gain, 23 life loss, 21 zone movement, 18 exile, 16 scry, and 8 stack
  counters. One definition may exercise more than one capability.
- Real translated examples passing include Swords to Plowshares, Counterspell,
  Nature's Claim, Sign in Blood, and Regrowth.
- The frozen 100-card T3.6 set currently has 17 runtime-smoke passes, 82 typed
  unsupported results, 0 failures, and one unmatched multiface identity. Its
  remaining leading blockers are ability shape (43), effect operation (17),
  keyword (17), and effect arguments (5).

The corpus report is generated at
`target/t3_5_runtime_interpreter_audit.json`; the stable evidence summary is in
`reports/gates/T3.5/runtime-interpreter-slice-2026-07-13.json`.

## Claim Boundary

A runtime-smoke pass proves that a supported compiled capability traverses
legal setup, targeting, payment, priority, production actions, zones, and
invariants. It is not card-specific semantic verification. All 100 frozen cards
remain unverified until T3.6 expected outcomes and deterministic replays pass.
T3.5 remains open while the vertical capability set needed by T3.6 and T3.9 is
implemented.
