# T3.3 Legacy API Mapper Specification

Status: in progress

## Contract

The mapper consumes positioned T3.2 legacy ability AST nodes and either emits
a typed Forge `Expression` plus typed costs or returns a stable quarantine
reason. It never substitutes prose or an approximate effect for an unsupported
legacy value.

Mapper registration is table-driven by the pair `(line prefix, legacy API)`.
Each registered mapper owns its accepted selector keys, parameter keys, value
grammar, target lowering, cost lowering, and structural test pack. Unknown
APIs, unknown parameters, duplicate parameters, missing required parameters,
dynamic values without an exact Forge representation, and unsupported target
languages all fail closed.

`metrics/api_coverage.json` reports each API's `legacy_uses`, `mapped`,
`verified`, and `quarantined` counts in descending corpus-frequency order.
`verified` currently means that the accepted structural mapping family has a
dedicated mapper test; it does not mean card-specific behavior has passed T3.6.
`metrics/api_quarantine.json` retains reason counts and deterministic samples.

## Initial Mapping Pack

The first pack targets exact high-frequency primitive cases:

- simple `A:... Mana` abilities with basic mana/tap costs and closed produced
  mana forms;
- simple `A:... Draw` abilities for the activating player with static positive
  card counts; and
- simple `A:... DealDamage` abilities with a positive literal amount and
  `Any` target language.

Complex costs, chained sub-abilities, dynamic amounts, replacement riders,
conditional execution, and narrower target predicates remain quarantined until
their value lowerers land. The initial pinned-corpus result maps 1,552 of
43,649 top-level ability uses (3.5556%) and quarantines the rest without
approximation.

The second pack adds exact static-integer Pump/PumpAll, life change, mill,
tap/untap, destroy, counter placement, and battlefield Continuous mappings.
Continuous mappings cover simple power/toughness modifiers and closed keyword
grants over self, aura/equipment, controller/owner, other-object, card-type,
and subtype selectors. Dynamic values, conditions, linked abilities, and
off-battlefield effects remain quarantined. The pinned-corpus result after this
pack maps 3,406 of 43,649 top-level ability uses (7.8032%).

The next packs continue in measured impact order, beginning with event APIs
whose Execute/SubAbility references require explicit SVar graph resolution.

The third pack adds script-level SVar graph resolution with stable diagnostics
for missing, duplicate, cyclic, malformed, and unsupported links. Exact ETB,
upkeep, attack, and spell-cast triggers can now resolve cost-free supported
effect chains. It also adds simple battlefield zone moves, token creation,
mass destroy/damage, discard, counterspell, scry, and surveil mappings. The
pinned-corpus result after this pack maps 6,102 of 43,649 top-level ability
uses (13.9797%) without approximating unsupported graph nodes.

The fourth pack records quarantine reasons per API and adds exact generic cost
reducers, simple damage triggers, fully resolved modal Charm graphs,
unconditional unblockability statics, and simple battlefield bulk zone moves.
The pinned-corpus result after this pack maps 6,486 of 43,649 top-level ability
uses (14.8594%).

## All-Confirmed-Blocker Planner

evaluates every top-level ability, every reachable SVar ability, and every
keyword independently. For the dominant `UNSUPPORTED_PARAMETER` class, the

are planning estimates only: actual coverage changes only when the normal
translator emits and compiler-roundtrips a card.
