# T3.4 Legacy Translation Factory

Status: verified locally

## Contract

`forge-porttools translate --all --jobs N` parses the pinned legacy corpus,
maps complete scripts through T3.3, emits canonical `.frs` files,
and compiler-roundtrips every emitted definition. A card is emitted only when
its catalog identity, printed characteristics, keywords, root abilities, SVar
links, and typed expressions all translate exactly. No partial card is written.
Ordered catalog face identities drive transform, split, adventure, modal, and
other multi-face emission, and each mapped ability remains on its source face.
Parameterized attachment and casting keywords are desugared into existing typed
abilities when the frozen DSL can retain their full costs and restrictions.
This currently covers common enchant, equip, fixed cycling and typecycling,
fixed enters-with-counters, and fixed alternative-cost keyword forms;
unsupported dynamic forms remain quarantined.

The generated directory is tool-owned and marked before it may be replaced.
Parallel workers perform read/parse/map/compile work; results and quarantine
records are sorted before deterministic sequential output. Ambiguous playable
catalog identities, parameterized keywords, metadata failures, mapper gaps, and
compiler failures receive stable file-level quarantine codes. Catalog-only and
out-of-v1 identities cannot be promoted into playable definitions.

The Owner-supplied `assets/coverage_priority.txt` is parsed as three strict,
unique tiers. Exact canonical names take precedence over face aliases, allowing
modal front-face requests to resolve without confusing reversible layouts.
`metrics/priority_coverage.json` records every requested card as emitted,
quarantined, missing from legacy data, ambiguous, or absent from the playable
catalog, including the exact blocker. Priority results guide mapper order but
do not substitute for corpus-wide or semantic coverage.

## Local Evidence

The 24-worker campaign considered 33,290 scripts, emitted 8,024 canonical
definitions (24.1033%), and quarantined 25,266 without partial output. It
resolved all 365 priority names and emitted 149 (40.8219%): 33/58 T0, 75/161
T1, and 35/146 T2. A repeat campaign produced the same complete output-tree
SHA-1 `c2f41ff326f4c6fcc470cff01ed89a02f6e0b3f6`. `forge-cardc` then built and
roundtripped all 8,024/8,024 definitions against 38,306 catalog identities and
113,234 printings. Thirty local porttools tests, clippy with warnings denied,
the recurring local verification suite, and workspace line coverage
23,338/28,977 (80.5397%) pass without network or GitHub Actions.
