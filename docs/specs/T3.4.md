# T3.4 Legacy Translation Factory

Status: verified locally

## Contract

`forge-porttools translate --all --jobs N` parses the pinned legacy corpus,
maps complete scripts through T3.3, emits canonical `.frs` files,
and compiler-roundtrips every emitted definition. A card is emitted only when
its catalog identity, printed characteristics, keywords, root abilities, SVar
links, and typed expressions all translate exactly. No partial card is written.
Ordered catalog face identities drive transform, split, adventure, modal, and
other multi-face emission, and each mapped ability remains on its source face.
Parameterized attachment and casting keywords are desugared into existing typed
abilities when the frozen DSL can retain their full costs and restrictions.
This currently covers common enchant, equip, fixed cycling and typecycling,
fixed enters-with-counters, and fixed alternative-cost keyword forms;
unsupported dynamic forms remain quarantined.

The generated directory is tool-owned and marked before it may be replaced.
Parallel workers perform read/parse/map/compile work; results and quarantine
records are sorted before deterministic sequential output. Ambiguous playable
catalog identities, parameterized keywords, metadata failures, mapper gaps, and
compiler failures receive stable file-level quarantine codes. Catalog-only and
out-of-v1 identities cannot be promoted into playable definitions.

The Owner-supplied `assets/coverage_priority.txt` is parsed as three strict,
unique tiers. Exact canonical names take precedence over face aliases, allowing
modal front-face requests to resolve without confusing reversible layouts.
`metrics/priority_coverage.json` records every requested card as emitted,
quarantined, missing from legacy data, ambiguous, or absent from the playable
catalog, including the exact blocker. Priority results guide mapper order but
do not substitute for corpus-wide or semantic coverage.

`translate --all` accepts explicit input/output paths and a strict
`--write-output true|false` control. Every run computes the same stable
dual-64-bit fingerprint over sorted emitted paths and canonical source. A
fingerprint-only run still performs parse, mapping, canonical emission in
memory, compiler roundtrip, quarantine classification, and priority reporting;
it only skips recreating thousands of `.frs` files.

`scripts/t3_parallel_sweep.sh development` overlaps the materializing
translation, blocker plan, mapping audit, and focused porttools tests, then
builds and validates the translated database. `checkpoint` gives the primary
sweep all 24 local workers, then overlaps a 12-worker fingerprint replay with
the compiler, 6-worker blocker planner, 1-worker planner replay, and mapping
audit. It compares translator and planner reports after normalizing only the
intentionally different worker count, requires byte-identical quarantine,
priority, and planner-detail reports, and then runs workspace fmt, clippy,
tests, and coverage. Rust entry points and the script reject more than 24
workers. Both modes run the Rayon boundary guard, force offline Cargo, and share
one cache, target, and generated output tree. A passing checkpoint regenerates
`STATUS.md`, `metrics/card_maturity.json`, and `metrics/project_status.json`.

## Local Evidence

The 24-worker campaign considered 33,290 scripts, emitted 8,700 canonical
definitions (26.1340%), and quarantined 24,590 without partial output. It
resolved all 365 priority names and emitted 173 (47.3973%): 35/58 T0, 84/161
T1, and 54/146 T2. Materialized and fingerprint-only replays matched output
fingerprint `392822b7a28413cec83d924fc970328e` plus byte-identical quarantine
and priority reports. `forge-cardc` built and roundtripped all 8,700/8,700
definitions against 38,306 catalog identities and 113,234 printings, and
`forge-cards` validated the database.

The accelerated core checkpoint completed in 37 seconds versus a measured
56-second serial baseline, saving 33.93%. The latest hardened checkpoint
completed in 76 seconds and passed 41 porttools tests, full workspace tests,
clippy with warnings denied, translation and blocker-planner deterministic
replay, compiler/database validation, and workspace line coverage
24,807/30,784 (80.5841%), without network or GitHub Actions. Generated evidence
lives in `metrics/t3_parallel_validation.json` and the truthful status artifacts
named above.
