# T3.4 Legacy Translation Factory

Status: verified locally

## Contract

`forge-porttools translate --all --jobs N` parses the pinned legacy corpus,
maps complete scripts through T3.3, emits canonical `.frs` files,
and compiler-roundtrips every emitted definition. A card is emitted only when
its catalog identity, printed characteristics, keywords, root abilities, SVar
links, and typed expressions all translate exactly. No partial card is written.
Ordered catalog face identities drive transform, split, adventure, modal, and
other multi-face emission, and each mapped ability remains on its source face.
Parameterized attachment and casting keywords are desugared into existing typed
abilities when the frozen DSL can retain their full costs and restrictions.
This currently covers common enchant, equip, fixed cycling, fixed enters-with-
counters, and fixed alternative-cost keyword forms; unsupported dynamic forms
remain quarantined.

The generated directory is tool-owned and marked before it may be replaced.
Parallel workers perform read/parse/map/compile work; results and quarantine
records are sorted before deterministic sequential output. Ambiguous playable
catalog identities, parameterized keywords, metadata failures, mapper gaps, and
compiler failures receive stable file-level quarantine codes. Catalog-only and
out-of-v1 identities cannot be promoted into playable definitions.

## Local Evidence

The 16-worker campaign considered 33,290 scripts, emitted 7,329 canonical
definitions (22.0156%), including 96 multi-face definitions, and quarantined
25,961 without partial output. A repeat
campaign produced the same complete output-tree SHA-1
`140c192f1f8427364cf4553248b647c27909eaf0`. `forge-cardc` then built and
roundtripped all 7,329/7,329 definitions against 38,306 catalog identities and
113,234 printings. Twenty-six local porttools tests, clippy with warnings
denied, 235 recurring oracles, and workspace line coverage 22,604/28,111
(80.4098%) pass without network or GitHub Actions.
