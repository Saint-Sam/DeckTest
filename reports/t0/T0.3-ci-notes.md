# T0.3 CI Notes

## Scope

- Added GitHub Actions CI under `.github/workflows/` only.
- Did not edit `Cargo.toml`, `scripts/`, `tools/`, `docs/adr`, `PLAN_STATE`, or `crates/`.

## Assumptions

- The root `Cargo.toml` and crate manifests are owned by T0.2. They appeared while this task was in progress, so the Cargo jobs are wired to run now; the bootstrap guards remain for scratch branches where T0.2 has not landed yet.
- The review/gate scripts are owned by T0.4. Existing executable hooks are called directly, and planned-but-missing recurring hooks skip until they land:
  - `scripts/vl.sh`
  - `scripts/review/determinism.sh`
  - `scripts/run_oracle.sh --all`
  - `scripts/fuzz_nightly.sh`
  - `scripts/card_regression.sh`
  - `tools/perf_diff.py`
- The biweekly interim drift review from section 4.3 is a human Gate Reviewer action, so it is not automated here.

## Problems

- I did not run Cargo locally because the task write scope only allowed `.github/workflows/` and this notes file; local Cargo commands could create `target/` or lock artifacts outside that scope.
- `scripts/run_oracle.sh`, `scripts/fuzz_nightly.sh`, and `scripts/card_regression.sh` are not present yet. The recurring workflow keeps the section 4.3 schedule registered and will run those commands once the scripts exist.
- The Android job assumes GitHub-hosted Ubuntu runners expose an Android NDK through `ANDROID_NDK_LATEST_HOME`. If that runner image changes, `build-android` may need an explicit setup action in a later CI hardening task.
